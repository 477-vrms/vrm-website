Main Goals for the Week

My main goals for the week were to get more familiar with Unity, and to be able to create movable arms within Unity. Since the arm relies on making a simulation for its movements, this task was top priority to me.

Unity

To get started with Unity and VR development, I did a couple tutorials teaching me how to interface the headset and controllers within the workspace. This also showed me a variety of features unity provides with object creation and managment.

Creating Robot Arms

After learning a bit about how VR development works within Unity, I started making a simple arm. For this arm, I followed some tutorials on Inverse Kinematics within Unity to figure out how to create this arm. I wrote two scripts, IKmanager.cs to calculate the overall Inverse Kinematics in 2D, and LookAt.cs, to track the position of the target and "look" at it, which allows the arm to track a target in 3D.

After doing that, I remade the arm to give it a more 3D look, as well as to sharpen my skills in creating arms.

These two arms seemed to work well, and will probably be used to test our base level arm since they have the same number of joints and rotation axis. However, one of my goals for the project was to create an arm simulation that allowed for more control, and these two didnt allow for much other than location tracking. What I wanted was rotational tracking in all axis as well. To do this, I decided to not spend time trying extremely complex inverse kinematics for multiple targets, so I instead looked to unity packages that can handle this part for me.
